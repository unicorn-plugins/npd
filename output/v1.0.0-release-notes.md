# v1.0.0 - 기획-설계-개발-배포 전 과정 지원 정식 릴리스

**릴리스 날짜**: 2026-03-06
**릴리스 유형**: Major

## 핵심 변경사항

- **기획/설계/개발/배포 4개 영역 전면 재작성**: clauding-guide 기준 플러그인 전면 개선, Phase 기반 병렬 워크플로우 도입
- **CI/CD 스킬 신규 추가**: Jenkins/GitHub Actions CI + ArgoCD CD 파이프라인 구축 지원
- **AI 엔지니어 통합**: 설계 워크플로우 전 단계에 ai-engineer 참여 반영
- **개발 품질 게이트 강화**: 독립 검증 단계, 자동화 테스트 체계, fail-fast 순서 재배치
- **3-Cloud 지원 확대**: AWS EKS, Azure AKS, GCP GKE 전 영역 가이드 보강

## 상세 변경사항

### 새로운 기능 (New Features)

- **CI/CD 스킬 추가**: Jenkins/GitHub Actions CI + ArgoCD CD 파이프라인 구축 워크플로우
- **AI 서비스 CI/CD 가이드 신규 작성**: AI 서비스 전용 빌드/배포 파이프라인
- **GCP GKE Autopilot 지원**: GKE Autopilot 섹션 추가 및 샘플 YAML 생성
- **AKS Automatic 지원**: Azure AKS Automatic 섹션 추가
- **EKS Auto Mode 지원**: StorageClass 및 ALB 설정 가이드
- **프론트엔드 개발 서버 실행기**: run-frontend-devserver.py 추가 및 전면 적용
- **커스텀 툴 추가**: run-intellij-service-profile.py
- **백킹서비스 가이드 확장**: RabbitMQ/Kafka 추가, 3-cloud 지원
- **deploy 백킹서비스 가이드 신설**: 컨테이너 네트워크 통합
- **CLAUDE.md 템플릿에 Lessons Learned 섹션 추가**: 스킬부스팅 규칙 및 시행착오 기록 체계

### 개선사항 (Improvements)

**기획 (Plan)**
- 기획 스킬 전면 개선: 에이전트 역할 강화, 가이드 보강, 프로토타입 병렬 개발 지원
- 기획 가이드 전면 개선: 구조 표준화, 내용 보강, 파일 재정리
- plan 스킬 Step 5/7 개선: 병렬 수행, 사용자 입력, 발산-수렴-선택 패턴 적용

**설계 (Design)**
- 설계 스킬 개선: Cloud 서비스 선택, 마이크로서비스별 아키텍처 패턴 선택, 병렬 처리 전략 범용화
- HighLevel 아키텍처/UI/UX 설계 가이드 보강 및 개발 표준 문서 추가
- AI 엔지니어 통합: 설계 워크플로우 전 단계에 ai-engineer 참여 반영
- 설계/개발/배포 가이드 표준화 및 SKILL.md 슬림화

**개발 (Develop)**
- 개발 스킬/가이드 전면 재작성: Phase 기반 병렬 워크플로우 도입
- 개발 가이드 대폭 개선: 프레임워크 중립 분기, 패키지 구조 표준 통일, OAuth2 인증 분기 추가
- 개발 품질 강화: TODO 금지, 런타임 검증 필수, 워크어라운드 금지 규칙 추가
- 개발 스킬 품질 게이트 강화: 독립 검증 단계 + 자동화 테스트 체계 도입
- 검증 게이트 전면 개편: fail-fast 순서 재배치, 오케스트레이터 보고 체계
- 개발 가이드 품질 개선: MCP Playwright 전환, Version Catalog 도입, CORS 강화
- develop 워크플로우 AI 개발 키트 체계 전환: P2 원칙 적용
- API 동기화 및 테스트 가이드 개선: 실제 코드 기반 패턴 탐색으로 전환
- 프론트엔드/백엔드-AI 연동 가이드에서 OpenAPI 명세 의존 제거: 실제 코드를 source of truth로 통일
- CORS 로컬 개발 지원: FRONTEND_PORT 변수 도입 및 전 가이드 반영

**배포 (Deploy)**
- 배포 가이드 대폭 개선: AI 서비스 추가, Flutter Web 통합, Cloud CLI 확장
- deploy 가이드 개선: AKS 정책 대응 및 Nginx 이중 도메인 설정
- K8s 배포 가이드 개선: 서비스별 리소스 설정, 환경변수 소스 통일, 입출력 경로 정리
- deploy Step 2/3에 PREV_ACTION/POST_ACTION 구조 도입 및 병렬 실행 지원
- 빌드 가이드 VM 기반 방식으로 전환: build-image-back/front/ai
- deploy 사전 준비 자동화: Step 1 신설 및 가이드 분리
- deploy Step 순서 변경: 물리 아키텍처 설계를 K8s 배포 직전으로 이동

**CI/CD**
- Jenkins CICD 가이드: Kaniko 인증/스토리지 설정 및 GCP Artifact Registry 지원 추가
- Jenkins CICD 가이드: AKS 전용 내용을 GKE에도 동일 적용
- ArgoCD 매니페스트 가이드: 누락 K8s 리소스 및 common/ 디렉토리 반영
- CI/CD 가이드 다수 개선: registry 기반 로그인 분기, IMG_NAME 추출, Kustomize 직접 다운로드
- SonarQube 버전 수정 및 Webhook 설정 가이드 개선
- ArgoCD Nginx 프록시에 WebSocket 및 타임아웃 설정 추가

**인프라 (Infrastructure)**
- K8s 생성 가이드 보강: EKS, AKS Automatic, GKE Autopilot
- Azure/GCP Ingress 아키텍처 다이어그램 추가
- 아키텍처 다이어그램 갱신: Nginx 프록시 + Public/Private Subnet 추가
- CLOUD CLI 설치 섹션 추가 (kubectl, helm, AWS/Azure/GCP CLI)
- AKS 비용절감: 시스템 노드풀 스케일링, az aks stop 미지원 반영

**플러그인 코어**
- AGENT.md 단순화 및 agentcard.yaml 역할 정보 강화
- plan/design/develop 스킬 EXPECTED OUTCOME에 GUIDE 기준 산출물 경로 추가
- .env 파일 루트 일원화 및 기타 가이드 정비
- runtime-env.js 동적 생성 가이드 통일 및 FRONTEND_PORT 변수 적용

### 문서 업데이트 (Documentation)
- Docker/K8s 레퍼런스 문서 추가
- k8s 참조 문서 보강, ALB 아키텍처 이미지, 샘플 산출물 추가

## 설치 및 사용

### 설치
```bash
claude mcp add-from-claude-plugin unicorn-plugins/npd
```

### 빠른 시작
```bash
# 도움말 보기
/npd:help

# 프로젝트 기획
/npd:plan

# 아키텍처 설계
/npd:design

# 개발
/npd:develop

# 배포
/npd:deploy

# CI/CD 파이프라인 구축
/npd:cicd

# GitHub Release 관리
/npd:ext-github-release-manager
```
