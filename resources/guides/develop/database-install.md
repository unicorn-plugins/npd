# 데이터베이스설치가이드

## 목적
제공된 설치 대상 환경에 데이터베이스설치계획서와 캐시설치계획서에 따라 데이터베이스 및 캐시를 병렬로 설치한다. 테이블은 생성하지 않고 데이터베이스만 설치하며, 설치 결과를 파일로 기록한다.

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| DB 설치 계획서 | `docs/develop/database-plan.md` | 설치 사양 확인 |
| 데이터 설계서 | `docs/design/database/{service-name}.md` | 스키마 기준 |

## 출력 (이 단계 산출물)

| 산출물 | 파일 경로 |
|--------|----------|
| DB 설치 실행 결과서 | `develop/database/exec/db-exec-{설치대상환경}.md` |
| 캐시 설치 실행 결과서 | `develop/database/exec/cache-exec-{설치대상환경}.md` |

## 방법론

- 제공된 {설치대상환경}에만 설치
- 데이터베이스설치계획서에 따라 병렬로 설치
  - (중요) 테이블은 생성하지 않고 데이터베이스만 설치
  - 기본 설치되는 데이터베이스 외에 추가 데이터베이스 생성하지 말것
- 캐시설치계획서에 따라 병렬로 설치
- 현재 OS에 맞게 설치
- 설치 후 데이터베이스 종류에 맞게 연결 방법 안내
- 설치 대상 클라우드 플랫폼은 이미 로그인되어 있고 Kubernetes도 연결되어 있음
- Kubernetes에 배포 manifest나 helm 설치를 위한 values.yaml은 develop/database/exec 디렉토리에 생성
- Database나 Redis를 Kubernetes에 배포한 경우 로컬에서 연결하기 위한 LoadBalancer 타입의 서비스 객체 생성
- 설치 결과 파일에는 설치 helm 명령, 설치 시 사용한 values.yaml 등을 포함하여 작성

## 출력 형식

- 파일명: `db-exec-{설치대상환경}.md`, `cache-exec-{설치대상환경}.md`
- `{설치대상환경}`은 `dev` 또는 `prod`로 함
- `{service-name}`은 영어로 작성
- 설치 helm 명령 및 사용한 values.yaml 내용 포함

## 품질 기준

- [ ] 환경변수 기반 설정
- [ ] 민감 정보 하드코딩 금지

## 주의사항

- 테이블은 생성하지 않고 데이터베이스(스키마)만 설치
- 기본 설치되는 데이터베이스 외에 추가 데이터베이스 생성 금지
- Kubernetes 배포 시 로컬 연결을 위한 LoadBalancer 서비스 객체 반드시 생성
