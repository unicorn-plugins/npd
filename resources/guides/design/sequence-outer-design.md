# 외부 시퀀스 설계 가이드

## 목적

유저스토리의 유저 플로우를 기반으로, 프론트엔드부터 백엔드 서비스·인프라·외부 시스템까지의 End-to-End 호출 흐름을 외부 시퀀스 다이어그램으로 설계함.

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| 유저스토리 | `docs/plan/design/userstory.md` | 요구사항과 유저 플로우('## 2. Epic 정의') 참조 |
| 논리 아키텍처 | `docs/design/logical-architecture.md` | 서비스 참여자 확인 |
| 사용자 플로우 프리젠테이션 | `docs/plan/design/uiux/prototype/flow-script.html` | 사용자 인터랙션 파악 |
| 시퀀스설계서(외부) 예제 | `{PLUGIN_DIR}/resources/samples/sample-시퀀스설계서(외부).puml` | 참조 |

## 방법론

### 작성 원칙
- **유저스토리와 매칭**되어야 함. **불필요한 추가 설계 금지**
- **논리 아키텍처에 정의한 참여자와 일치**해야 함
- **해당 유저 플로우에 참여하는** 프론트엔드, 모든 서비스, 인프라 컴포넌트(예: Gateway, Message Queue, Database), 외부 시스템을 참여자로 추가
- 유저 플로우에 참여자들의 역할과 책임을 명확히 표시
- 유저 플로우 수행에 필요한 프론트엔드부터 End-to-End 호출을 순서대로 표시하고 한글 설명 추가
- 마이크로서비스 내부의 처리 흐름은 표시하지 않음
- 동기/비동기 통신 구분 (실선/점선)
- 캐시, 큐 등 인프라 컴포넌트와의 상호작용 포함
- **AI 서비스가 관여하는 플로우**에서는 AI Pipeline 서비스를 참여자로 반드시 포함
- AI 외부 API 호출(예: LLM API)은 비동기 통신(점선)으로 표현하고, 타임아웃/폴백 흐름을 alt 블록으로 표시

### 작성 순서

- 준비:
  - 유저스토리, 논리 아키텍처 분석 및 이해
  - 유저스토리의 '## 2. Epic 정의'섹션에 유저 플로우 파악 
  - 시퀀스설계서(외부) 예제 참조
  - 사용자 플로우 프리젠테이션을 웹브라우저에서 실행하여 서비스 이해
- 실행:
  - 핵심 비즈니스 플로우별로 파일을 분리하여 작성
  - 병렬 수행 가이드에 따라 병렬 수행
  - **PlantUML 스크립트 파일 생성 즉시 검사 실행**
- 검토:
  - 작성 원칙 준수 검토

### 병렬 수행

- **서브 에이전트를 활용한 병렬 작성 필수**
- 서비스별 독립적인 에이전트가 각 외부 시퀀스 설계를 동시에 작업
- 모든 설계 완료 후 전체 검증

## 출력 형식

```plantuml
!theme mono

title {유저 플로우명}

actor "사용자" as user
participant "프론트엔드" as fe
participant "{서비스명}" as svc
participant "{인프라}" as infra
participant "AI Pipeline" as ai
participant "LLM API(E)" as llm

user -> fe : {액션 설명}
fe -> svc : {API 호출 설명}
svc --> fe : {응답 설명}
fe --> user : {화면 표시 설명}
```
