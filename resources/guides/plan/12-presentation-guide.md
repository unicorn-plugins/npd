# 서비스 기획서 발표자료 스크립트 작성 가이드

## 목적

비즈니스 모델을 기반으로 서비스 기획서 발표자료 스크립트를 작성함.
- gamma.app 등 프레젠테이션 도구로 변환하기 위한 마크다운 스크립트 생성
- 문제 → 솔루션 → 가치의 스토리텔링 흐름으로 구성

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| 비즈니스 모델 | `docs/plan/think/비즈니스모델.md` | Lean Canvas 9영역 + 경쟁 분석 + GTM + 재무 계획을 장표로 변환 |

## 출력 (이 단계 산출물)

| 산출물 | 파일 경로 |
|--------|----------|
| 서비스 기획서 발표자료 스크립트 | `docs/plan/think/서비스기획서스크립트.md` |

## 방법론

### 발표자료 표준 구조 (10-15장)

린캔버스 9가지 영역을 중심으로 구성합니다.

#### 린캔버스 9영역
1. **Problem** (문제)
2. **Customer Segments** (고객 세그먼트)
3. **Unique Value Proposition** (고유 가치 제안)
4. **Solution** (솔루션)
5. **Channels** (채널)
6. **Revenue Streams** (수익 흐름)
7. **Cost Structure** (비용 구조)
8. **Key Metrics** (핵심 지표 — AARRR)
9. **Unfair Advantage** (경쟁우위)

#### 추가 분석
10. **경쟁 분석 및 포지셔닝**
11. **Go-to-Market 전략** (Pre-launch / Launch / Post-launch)
12. **재무 계획** (3개년 손익계산서, BEP)

### 스크립트 작성 원칙

| 원칙 | 설명 |
|------|------|
| **간결성** | 각 슬라이드당 핵심 메시지 3개 이하 |
| **스토리텔링** | 문제 → 솔루션 → 가치의 논리적 흐름 |
| **비주얼 중심** | 텍스트 최소화, 비주얼(차트·그래프·다이어그램)로 설명 |
| **데이터 기반** | 주장을 정량 데이터로 뒷받침 |
| **슬라이드 구분** | 각 슬라이드 사이를 `---`로 구분 |

## 출력 형식

```markdown
# 서비스 기획서 발표자료 스크립트

## {장표 제목}

**핵심 메시지**:
- {메시지 1}
- {메시지 2}
- {메시지 3}

**비주얼 제안**: {차트/그래프/다이어그램 설명}

**발표 노트**: {발표자가 말할 내용 요약}

---

(10-15장 반복)
```

## 품질 기준

### 완료 체크리스트
- [ ] 10-15장 슬라이드 구성
- [ ] 장표당 핵심 메시지 3개 이하
- [ ] 문제 → 솔루션 → 가치의 스토리텔링 흐름 유지
- [ ] 비주얼 제안 포함 (차트/그래프/다이어그램)
- [ ] 데이터 기반 주장 뒷받침
- [ ] 장표 간 `---` 구분
- [ ] 비즈니스모델.md의 핵심 내용이 누락 없이 반영됨

### 정량 기준
- 슬라이드: 10-15장
- 장표당 핵심 메시지: 3개 이하
- 비주얼 제안: 장표당 1개 이상

## 주의사항

- 슬라이드당 텍스트 과다 금지
- 핵심 메시지 3개 초과 금지
- 데이터 없는 추상적 주장 금지
- 장표 간 논리적 흐름이 끊기지 않도록 구성
