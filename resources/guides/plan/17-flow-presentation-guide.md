# 사용자 플로우 프리젠테이션 가이드

## 목적

프로토타입 화면 스크린샷을 캡처하고, 사용자 플로우 순서대로 화면을 보여주는
인터랙티브 HTML 프리젠테이션을 개발함.
- 프로토타입의 실제 화면을 3-5개 주요 사용자 플로우로 구성
- 서비스의 핵심 사용 시나리오를 시각적으로 전달

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| 통합 테스트 결과 | `docs/plan/design/uiux/prototype/테스트결과.md` | 화면간 연결성 데이터를 분석하여 사용자 플로우 도출 |
| UI/UX 설계 | `docs/plan/design/uiux/uiux.md` | 사용자 플로우 정의, 화면 순서 참조 |
| 프로토타입 화면 | `docs/plan/design/uiux/prototype/*.html` | Playwright 스크린샷 캡처 대상 |

## 방법론

### 1. 화면 캡처

Playwright를 사용하여 각 프로토타입 화면에 접속하여 스크린샷을 촬영함.

#### 캡처 사양
- **해상도**: 375x812 (모바일 기준)
- **포맷**: PNG
- **파일명**: `{순번}-{화면명}.png`
- **저장 경로**: `docs/plan/design/uiux/prototype/screenshots/`

#### 자막 오버레이 스타일
각 이미지에 일련번호와 한글 자막을 주입함:
- **위치**: 하단 고정
- **구분선**: 빨간 구분선 (height: 3px, color: #FF3B30)
- **배경**: 다크 그라디언트 (rgba(0,0,0,0.85))
- **제목**: 흰색 16px bold
- **설명**: 회색 14px (#AAAAAA)

### 2. 플로우 구성

테스트결과.md의 화면간 연결성 데이터를 분석하여 주요 사용자 플로우를 도출함.

#### 플로우 도출 기준
- 테스트결과.md의 "화면간 연결성" 테이블에서 화면 간 이동 경로를 추출
- 3-5개 주요 사용자 플로우를 식별:
  - **신규 온보딩 플로우**: 회원가입 → 초기 설정 → 메인 화면
  - **핵심 기능 수행 플로우**: 메인 → 핵심 기능 사용 → 결과 확인
  - **설정/관리 플로우**: 마이페이지 → 설정 → 프로필 관리
  - 기타 서비스 특화 플로우

### 3. 프리젠테이션 개발

#### 3단 레이아웃 상세

| 영역 | 너비 | 내용 |
|------|------|------|
| 좌측 (스텝 리스트) | 250px | 현재 플로우의 스텝 목록, 방문 체크마크, 프로그레스 바 |
| 중앙 (폰 프레임) | 가변 | 스크린샷을 폰 프레임 안에 표시 |
| 우측 (나레이션 패널) | 300px | 현재 스텝의 사용자 액션 설명 + 나레이션 텍스트 |

#### 인터랙션 기능
- **플로우 탭 전환**: 상단 탭으로 플로우 간 전환
- **키보드 네비게이션**: 좌(←)/우(→) 화살표로 스텝 이동
- **클릭 네비게이션**: 스텝 리스트 클릭으로 직접 이동
- **진행 추적**: 방문한 스텝에 체크마크 표시, 프로그레스 바

#### 반응형 대응

| 뷰포트 | 레이아웃 |
|--------|---------|
| 데스크톱 (> 1024px) | 3단 (스텝리스트 + 폰프레임 + 나레이션) |
| 태블릿 (768-1024px) | 2단 (폰프레임 + 나레이션 하단) |
| 모바일 (< 768px) | 1단 (스텝리스트 접힘, 폰프레임 + 나레이션 하단) |

#### 테마
- 프로토타입과 동일한 다크 테마 적용

## 출력 형식

### screenshots/ 디렉토리 구조
```
docs/plan/design/uiux/prototype/screenshots/
├── 01-{화면명}.png
├── 02-{화면명}.png
├── ...
└── {N}-{화면명}.png
```

### flow-script.html 구조
```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>{서비스명} - 사용자 플로우</title>
  <style>
    /* 다크 테마, 3단 레이아웃, 반응형 */
  </style>
</head>
<body>
  <!-- 플로우 탭 -->
  <nav class="flow-tabs">
    <button class="flow-tab active" data-flow="0">{플로우1명}</button>
    <button class="flow-tab" data-flow="1">{플로우2명}</button>
  </nav>

  <div class="layout">
    <!-- 좌측: 스텝 리스트 -->
    <aside class="step-list">...</aside>
    <!-- 중앙: 폰 프레임 -->
    <main class="phone-frame">
      <img src="screenshots/{이미지}" alt="{화면명}">
    </main>
    <!-- 우측: 나레이션 -->
    <aside class="narration">...</aside>
  </div>

  <script>
    // 플로우 데이터, 키보드 네비게이션, 진행 추적
    const flows = [
      {
        name: "{플로우명}",
        steps: [
          { image: "screenshots/{파일명}.png", title: "{화면명}", action: "{사용자 액션}", narration: "{나레이션}" }
        ]
      }
    ];
  </script>
</body>
</html>
```

## 품질 기준

### 완료 체크리스트
- [ ] 프로토타입의 모든 화면 스크린샷 캡처 완료
- [ ] 각 스크린샷에 자막 오버레이 적용 (번호 + 화면명 + 설명)
- [ ] 3-5개 사용자 플로우 구성
- [ ] 테스트결과.md의 화면간 연결성 데이터 기반 플로우 구성
- [ ] 3단 레이아웃 (스텝 리스트 / 폰 프레임 / 나레이션) 구현
- [ ] 키보드(←/→) 네비게이션 동작
- [ ] 플로우 탭 전환 동작
- [ ] 반응형 대응 (데스크톱 3단 / 태블릿 2단 / 모바일 1단)
- [ ] 다크 테마 적용
- [ ] Playwright 검증 통과 (정상 로드, 네비게이션, 스크린샷 표시)

### 정량 기준
- 사용자 플로우: 3-5개
- 스크린샷: 프로토타입 전체 화면 수
- 반응형 뷰포트: 3종 (데스크톱/태블릿/모바일)

## 주의사항

- 프로토타입에 없는 화면을 포함하지 않을 것
- 화면간 연결성 데이터에 없는 플로우를 임의로 만들지 않을 것
- 프로토타입과 동일한 다크 테마 적용
- 스크린샷은 모바일 해상도(375x812) 기준
- 자막 오버레이는 스크린샷 이미지 위에 CSS로 오버레이 (원본 이미지 비파괴)
