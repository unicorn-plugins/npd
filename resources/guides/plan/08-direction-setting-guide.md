# 방향성 정의 가이드 (킹핀 문제 + Needs Statement)

## 목적

검증된 문제들 중에서 가장 핵심적인 킹핀 문제를 찾고,
이를 해결하기 위한 명확한 방향성을 Needs Statement로 정의함.

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| 문제 가설 | `docs/plan/define/문제가설.md` | 검증된 문제 리스트업 및 인과관계 분석에 활용 |
| 문제검증 인터뷰 결과 취합 | `docs/plan/define/문제검증인터뷰결과취합.md` | 문제별 중요도/불편도 데이터로 평가에 활용 |
| 비즈니스 가치 | `docs/plan/define/비즈니스가치.md` | 킹핀 문제 선정 시 가치 연결 확인 |
| 고객 분석 | `docs/plan/define/고객분석.md` | Needs Statement의 고객유형 정의에 활용 |

## 방법론

### 킹핀 문제 개념

볼링에서 스트라이크를 치기 위해 반드시 쓰러뜨려야 하는 5번 핀처럼,
해결하면 다른 여러 문제들이 연쇄적으로 해결되는 가장 핵심적인 문제임.

### 5가지 선정 기준 (각 5점)

| 기준 | 핵심 질문 | 평가 포인트 |
|------|---------|-----------|
| **영향력 (Impact)** | 해결 시 다른 문제도 함께 해결되는가? | 연쇄 해결 문제의 수와 중요도 |
| **빈도 (Frequency)** | 고객이 얼마나 자주 겪는가? | 매일/매주/매월/가끔 |
| **심각도 (Severity)** | 고객에게 미치는 고통의 정도는? | 비용·시간 손실·감정적 고통 |
| **근본성 (Root Cause)** | 다른 문제들의 원인이 되는가? | 5WHY 분석 결과, 인과관계 |
| **해결가능성 (Feasibility)** | 현실적으로 해결 가능한가? | 기술·자원·시간·비용 |

### 평가 기준 (Rubric)

| 점수 | 영향력 | 빈도 | 심각도 | 근본성 | 해결가능성 |
|------|--------|------|--------|--------|-----------|
| 1 | 단일 사용자 일부 경험에만 영향 | 연 1-2회 발생 | 사소한 불편 | 표면적 증상 | 기술적 불가능 |
| 2 | 일부 사용자 일부 경험에 영향 | 분기 1회 발생 | 불편하지만 대안 존재 | 하위 문제 1개의 원인 | 12개월 이상 소요 |
| 3 | 다수 사용자 일부 경험에 영향 | 월 1-2회 발생 | 작업 효율 저하 | 하위 문제 2-3개의 원인 | 6-12개월 소요 |
| 4 | 다수 사용자 핵심 경험에 영향 | 주 1-2회 발생 | 대안 탐색 유발 | 다수 문제의 공통 원인 | 3-6개월 소요 |
| 5 | 전체 사용자 핵심 경험에 영향 | 매일 발생 | 서비스 이탈 유발 | 거의 모든 문제의 근본 원인 | 즉시 해결 가능 |

### 분석 방법

#### Step 1: 검증된 문제 리스트업
| 문제 ID | 문제 설명 | 중요도 평균 | 불편도 평균 |
|---------|-----------|-------------|-------------|
| P1 | {문제} | X.X | X.X |

#### Step 2: 문제 간 인과관계 맵핑
```
P1 (근본원인) → P2 (중간문제) → P3 (표면문제)
```

#### Step 3: 5가지 기준 평가
| 문제 ID | 영향력 | 빈도 | 심각도 | 근본성 | 해결가능성 | 총점 |
|---------|--------|------|--------|--------|------------|------|

#### Step 4: 종합 분석 및 킹핀 문제 선정

### Needs Statement 작성

#### 구조
```
'<고객유형>는 <목적>을 위하여 <원하는 것>이 필요하다.'
```

#### 작성 원칙

| 원칙 | 설명 |
|------|------|
| **사용자 경험 중심** | 기술적 솔루션이 아닌 사용자 경험의 향상 표현 |
| **목적 명확화** | 킹핀 문제의 근본원인을 제거하는 것을 목적으로 설정 |
| **원하는 것 정의** | 기능이 아닌 경험 표현. 큰 방향성 또는 컨셉 정의 |
| **하나의 문장** | 여러 개가 아닌 단 1개의 문장으로 표현 |

#### 예시

```
'가성비 좋은 생활 구독서비스를 이용하고 싶은 구독서비스 이용자는
복잡한 구독을 효율적으로 관리하기 위하여
모든 구독을 한눈에 파악하고 최적화할 수 있는 통합 관리 경험이 필요하다.'
```

#### 검증 체크리스트
- [ ] 고객유형이 명확하게 정의되어 있는가?
- [ ] 목적이 킹핀 문제의 근본원인 해결과 연결되어 있는가?
- [ ] 원하는 것이 기능이 아닌 경험으로 표현되었는가?
- [ ] 하나의 문장으로 명확하게 표현되었는가?
- [ ] 솔루션이 아닌 방향성을 제시하는가?

## 품질 기준

### 완료 체크리스트
- [ ] 5가지 기준(영향력/빈도/심각도/근본성/해결가능성) 평가 완료
- [ ] 킹핀 문제 1개 선정 (선정 근거 포함)
- [ ] Needs Statement 1문장 작성
- [ ] Needs Statement가 기능이 아닌 경험으로 표현됨
- [ ] 문제 간 인과관계 분석 포함
- [ ] 문제가설.md, 문제검증인터뷰결과취합.md 데이터 기반

### 정량 기준
- 평가 기준: 5가지 (각 5점 만점)
- 킹핀 문제: 1개
- Needs Statement: 1문장

## 주의사항

- 킹핀 문제는 객관적 데이터 기반 결정
- Needs Statement는 솔루션이 아닌 방향성 제시
- 감정적 판단이 아닌 객관적 데이터로 결정
- 킹핀 문제 해결이 다른 문제들의 연쇄 해결로 이어지는지 검증
