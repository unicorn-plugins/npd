# 고객 분석 가이드 (JTBD)

## 목적

JTBD(Jobs To Be Done) 프레임워크를 사용하여 타겟 고객 유형을 정의함.

## 분석 프레임워크

### 타겟 고객 유형 (JTBD 형식)
`[사용자 세그먼트] + [달성하려는 Job]`

**예시:**
- 집밥을 먹고 싶은 음식 소비자
- 비용 효율적인 생활 구독을 원하는 구독 서비스 이용자
- 맞춤형 여행 일정을 계획하고 싶은 여행 계획자

### 사용자 세그먼트 (3-5개)
각 세그먼트별 정의 항목:
- **세그먼트 명**: 구체적인 사용자 그룹명
- **인구통계학적 특성**: 연령, 직업, 소득, 지역 등
- **행동 패턴**: 습관, 소비 패턴, 정보 탐색 방식
- **현재 상황**: 현재 어떤 방식으로 문제를 해결하고 있는지

### Job to be Done
- **주요 Job**: 고객이 달성하고자 하는 핵심 목표
- **관련 Jobs**: 주요 Job과 연관된 부수적 목표
- **원하는 결과**: 고객이 기대하는 최종 성과

## 출력 형식

```markdown
# 고객 분석

## MVP 주제
{MVP 주제}

## 타겟 고객 유형 (JTBD)
{사용자 세그먼트} + {달성하려는 Job}

## 사용자 세그먼트

### 세그먼트 1: {세그먼트명}
- **인구통계**: {설명}
- **행동 패턴**: {설명}
- **현재 상황**: {설명}

(3-5개 세그먼트 반복)

## Job to be Done

### 주요 Job
{고객이 달성하려는 주요 목표}

### 관련 Jobs
- {관련 Job 1}
- {관련 Job 2}

### 원하는 결과
- {결과 1}
- {결과 2}
```

## 주의사항

- JTBD는 "무엇을 달성하려는가"에 집중 (기능이 아님)
- 고객 관점에서 작성 (기업 관점 X)
- 구체적이고 측정 가능한 표현 사용
- 3-5개의 명확한 세그먼트로 구분
- 각 세그먼트의 차별화된 특성 명시
- 솔루션이 아닌 고객 목표에 집중
