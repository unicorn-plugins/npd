# 고객 분석 가이드 (JTBD)

## 목적

JTBD(Jobs To Be Done) 프레임워크를 사용하여 타겟 고객 유형을 정의함.

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| MVP 정의 | `docs/plan/define/MVP정의.md` | MVP 주제와 도메인을 기반으로 분석 범위 설정 |

## 방법론

### 타겟 고객 유형 (JTBD 형식)
`[사용자 세그먼트] + [달성하려는 Job]`

**예시:**
- 집밥을 먹고 싶은 음식 소비자
- 비용 효율적인 생활 구독을 원하는 구독 서비스 이용자
- 맞춤형 여행 일정을 계획하고 싶은 여행 계획자

### 사용자 세그먼트 (3-5개)
각 세그먼트별 정의 항목:
- **세그먼트 명**: 구체적인 사용자 그룹명
- **인구통계학적 특성**: 연령, 직업, 소득, 지역 등
- **행동 패턴**: 습관, 소비 패턴, 정보 탐색 방식
- **현재 상황**: 현재 어떤 방식으로 문제를 해결하고 있는지

### Job to be Done
- **주요 Job**: 고객이 달성하고자 하는 핵심 목표
- **관련 Jobs**: 주요 Job과 연관된 부수적 목표
- **원하는 결과**: 고객이 기대하는 최종 성과

### Job 유형 분류

각 Job을 아래 3가지 유형으로 분류하여 분석함:

| Job 유형 | 설명 | 예시 |
|---------|------|------|
| **Functional Job** | 실용적/기능적 목표 | "매주 식단을 계획하고 싶다" |
| **Emotional Job** | 감정적 목표 | "건강한 식습관에 대한 자신감을 느끼고 싶다" |
| **Social Job** | 사회적 목표 | "가족에게 건강을 챙기는 사람으로 인정받고 싶다" |

### 세그먼트 우선순위 결정

| 기준 | 설명 |
|------|------|
| 시장 크기 | 해당 세그먼트의 잠재 사용자 수 |
| 접근 용이성 | 마케팅/채널 도달 가능성 |
| Pain Point 심각도 | 문제의 시급성 |
| 지불 의향 | 수익 잠재력 |
| 경쟁 강도 | 해당 세그먼트에 대한 기존 솔루션 수 |

## 출력 형식

```markdown
# 고객 분석

## MVP 주제
{MVP 주제}

## 타겟 고객 유형 (JTBD)
{사용자 세그먼트} + {달성하려는 Job}

## 사용자 세그먼트

### 세그먼트 1: {세그먼트명}
- **인구통계**: {설명}
- **행동 패턴**: {설명}
- **현재 상황**: {설명}

(3-5개 세그먼트 반복)

## Job to be Done

### 주요 Job
{고객이 달성하려는 주요 목표}

### 관련 Jobs
- {관련 Job 1}
- {관련 Job 2}

### 원하는 결과
- {결과 1}
- {결과 2}
```

## 품질 기준

### 완료 체크리스트
- [ ] 사용자 세그먼트 3-5개 식별
- [ ] 각 세그먼트별 인구통계/행동패턴/현재상황 모두 포함
- [ ] JTBD 형식으로 작성 (사용자 세그먼트 + 달성하려는 Job)
- [ ] Job 유형(Functional/Emotional/Social) 분류 포함
- [ ] 솔루션이 아닌 고객 목표에 집중하여 작성됨
- [ ] MVP정의.md와 일관성 확인

### 정량 기준
- 사용자 세그먼트: 3-5개
- 각 세그먼트: 인구통계 + 행동 패턴 + 현재 상황 3가지 필수
- 관련 Jobs: 2개 이상

## 주의사항

- JTBD는 "무엇을 달성하려는가"에 집중 (기능이 아님)
- 고객 관점에서 작성 (기업 관점 X)
- 구체적이고 측정 가능한 표현 사용
- 3-5개의 명확한 세그먼트로 구분
- 각 세그먼트의 차별화된 특성 명시
- 솔루션이 아닌 고객 목표에 집중
