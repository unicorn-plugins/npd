# 고객 여정 맵 작성 가이드

## 목적

고객의 여정을 시각화하고 분석하는 User Journey Map을 작성함.

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| 고객경험 단계 | `docs/plan/define/고객경험단계.md` | 여정 맵의 X축으로 사용 |
| 관찰 결과 | `docs/plan/define/관찰결과.md` | 단계별 행동/어려움 데이터를 여정 맵에 반영 |
| 체험 결과 | `docs/plan/define/체험결과.md` | 단계별 경험/만족도 데이터를 여정 맵에 반영 |
| 고객경험 인터뷰 결과 | `docs/plan/define/고객경험인터뷰결과.md` | 단계별 행동/생각/감정 데이터를 여정 맵에 반영 |
| 인터뷰 결과 취합 | `docs/plan/define/고객경험인터뷰결과취합.md` | 핵심 인사이트와 Pain Points를 여정 맵에 반영 |

## 출력 (이 단계 산출물)

| 산출물 | 파일 경로 |
|--------|----------|
| 고객 여정 맵 | `docs/plan/define/유저저니맵.md` |
| 고객 여정 맵 시각화 | `docs/plan/define/유저저니맵.svg` |

## 방법론

### 1. 페르소나
대표 사용자를 정의함:
- 이름, 나이, 직업
- 배경 및 상황
- 목표 및 동기
- 기술 숙련도

### 2. 고객경험 단계 (X축)
customer-journey-stages에서 정의한 고객경험 단계를 X축으로 사용함.

### 3. 단계별 분석 항목

각 단계를 다음 요소로 분석함:
- **고객 행동**: 무엇을 하는가?
- **생각**: 무엇을 생각하는가?
- **감정**: 어떻게 느끼는가? (감정 곡선)
- **터치포인트**: 어디에서 서비스와 상호작용하는가?
- **Pain Points**: 어려움과 장애물
- **Gain Points**: 긍정적 경험과 만족 요소

### 4. 핵심 인사이트
- 가장 큰 Pain Points 상위 3개
- 개선 기회 영역
- 감정의 고점과 저점
- 이탈 위험 지점

### 5. 기회 영역
- 높은 우선순위 개선 영역
- 혁신이 필요한 터치포인트
- 차별화 포인트

### 감정 레벨 기준 (5점)

| 레벨 | 감정 상태 | 설명 |
|------|----------|------|
| 1 | 극도로 불편/좌절 | 서비스 이탈을 고려하는 수준의 부정적 경험 |
| 2 | 불편/불만족 | 명확한 불편함을 느끼지만 참을 수 있는 수준 |
| 3 | 중립/무관심 | 특별한 감정 없이 기능적으로만 사용 |
| 4 | 만족/긍정적 | 기대에 부합하는 긍정적 경험 |
| 5 | 매우 만족/감동 | 기대를 초과하는 감동적 경험 |

## 감정 곡선 표현

```
 높음 ┤     ╭─╮
      │    ╱   ╲    ╭─╮
      │   ╱     ╲  ╱   ╲
 중간 ┤  ╱       ╲╱     ╰─╮
      │ ╱                  ╲
 낮음 ┤╯                    ╰
      └─────────────────────────>
       단계1  단계2  단계3  단계4  단계5
```

## 시각화 가이드라인

- 감정 상태 표현: 긍정 / 중립 / 부정
- 감정 레벨: 5점 만점
- 명확한 단계 경계 사용
- 중요한 순간 강조
- 시간 경과에 따른 감정 궤적 표시

## 출력 형식

```markdown
# 고객 여정 맵

## 1. 페르소나 정의

- **이름**: {가명}
- **세그먼트**: {고객분석의 세그먼트}
- **배경**: {연령, 직업, 상황}
- **목표**: {달성하려는 Job}
- **좌절 요소**: {주요 Pain Point}

## 2. 여정 맵 상세

| 고객경험 단계 | 행동 | 생각 | 감정 (1-5) | 터치포인트 | Pain Points | Gain Points |
|-------------|------|------|-----------|-----------|------------|------------|
| {단계1} | {행동} | {생각} | {점수} | {접점} | {문제} | {긍정} |

## 3. 감정 곡선

{ASCII 시각화}

## 4. 핵심 인사이트
- {인사이트 1}
- {인사이트 2}
- {인사이트 3}

## 5. 기회 영역
- {기회 1}: {설명}
- {기회 2}: {설명}
- {기회 3}: {설명}
```

## 품질 기준

### 완료 체크리스트
- [ ] 페르소나 정의 포함 (이름, 세그먼트, 배경, 목표, 좌절 요소)
- [ ] 고객경험 단계를 X축으로 사용
- [ ] 각 단계별 행동/생각/감정/터치포인트/Pain Points/Gain Points 작성
- [ ] 감정 곡선 포함 (1-5점 기준)
- [ ] 핵심 인사이트 도출
- [ ] 기회 영역 3개 이상 도출
- [ ] SVG 파일 시각화 포함
- [ ] 고객 경험 조사 데이터 기반 작성

### 정량 기준
- 기회 영역: 3개 이상
- 감정 레벨: 5점 기준
- Pain Points: 상위 3개 이상

## 주의사항

- 고객경험 단계를 X축으로 사용
- 실제 고객 데이터 기반 작성 (추측 X)
- 감정 곡선은 정량적 데이터 기반
- Pain Point는 구체적이고 측정 가능하게
- 터치포인트는 실제 접점 위주로
- 개선 기회는 실행 가능하게
- SVG 파일로 시각화 포함
