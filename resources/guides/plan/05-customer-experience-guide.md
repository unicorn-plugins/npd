# 고객 경험 조사 가이드

## 목적

인터뷰, 관찰, 체험이라는 3가지 조사 방법을 통해 포괄적인 고객 경험 데이터를 생성함.

> **참고**: 본 조사는 AI 에이전트가 시뮬레이션하는 가상 데이터입니다. 실제 고객 조사를 대체하지 않으며, 서비스 기획 초기 단계에서 가설을 수립하기 위한 참고 자료로 활용됩니다.

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| 고객 분석 | `docs/plan/define/고객분석.md` | 세그먼트 정보를 기반으로 조사 대상자 프로필 구성 |
| 고객경험 단계 | `docs/plan/define/고객경험단계.md` | 경험 단계를 X축 기준으로 데이터 구조화 |

## 출력 (이 단계 산출물)

| 산출물 | 파일 경로 |
|--------|----------|

| 고객경험 인터뷰 결과 | `docs/plan/define/고객경험인터뷰결과.md` |

## 방법론

### 1. 고객 경험 인터뷰

#### 인터뷰 개요
- **목적**: 현재 환경에서의 고객 행동, 인식, 감정 반응 및 불편 사항 분석을 통한 현재 문제와 잠재 니즈 발굴
- **방법**: 1:1 심층 인터뷰

#### 인터뷰 일정 형식
| 인터뷰 일시 | 장소 | 인터뷰어 | 고객명 | 고객 프로필 |
|------------|------|----------|--------|------------|
| {날짜/시간} | {장소} | {인터뷰어} | {가명} | {나이, 직업, 특징} |

#### 고객별 인터뷰 내용 형식

**고객 N: {가명} - {프로필}**

**{고객경험 단계 N}**
- **행동**: {이 단계에서 하는 행동}
- **생각**: {이 단계에서의 생각}
- **긍정적 느낌**: {좋은 점}
- **부정적 느낌**: {불편한 점, 문제}

(모든 고객경험 단계 반복)

#### 핵심 인사이트
- **Pain Points**: 주요 문제점 3-5개
- **Needs**: 핵심 니즈 3-5개
- **기대 가치**: 고객이 기대하는 가치
- **주목할 만한 인용구**: 고객의 실제 발언

### 2. 고객 관찰 결과

#### 관찰 개요
- **목적**: 현재 환경에서의 고객 행동, 인식, 감정 반응 및 불편 사항 분석
- **방법**: 현장 관찰, Contextual Inquiry

#### 관찰 결과 형식

**관찰 N: {가명} - {프로필}**

**{고객경험 단계 N}**
- **관찰된 행동**: {구체적 행동 기술}
- **어려움/문제**: {발견된 문제}
- **특이사항**: {주목할 점}

(모든 고객경험 단계 반복)

#### Pain Point 및 니즈 분석
| 관찰된 문제 | 도출된 니즈 |
|-----------|-----------|
| {문제} | {니즈} |

#### 행동 패턴 분석
- **패턴 N**: {설명}
  - 빈도: {높음/중간/낮음}
  - 시사점: {의미}

### 3. 고객 체험 테스트

#### 체험 개요
- **목적**: 기존 서비스/제품 사용 체험을 통한 문제점 및 개선 기회 발견
- **방법**: 실제 사용 체험

#### 상세 체험 평가 형식

**체험 N: {서비스/제품명}**

**{고객경험 단계 N}**
- **경험 내용**: {구체적 경험}
- **긍정적 측면**: {좋았던 점}
- **부정적 측면**: {불편했던 점}
- **만족도**: {5점 만점}

(모든 고객경험 단계 반복)

## 품질 기준

### 완료 체크리스트
- [ ] 사용자 입력 규모(인터뷰수/관찰수/체험수) 충족
- [ ] 고객경험 단계별로 데이터 구조화
- [ ] 개별 결과 누락 없이 모두 제공
- [ ] Pain Point와 니즈가 명확히 구분됨
- [ ] 고객분석.md, 고객경험단계.md와 일관성 확인

### 정량 기준
- 인터뷰 결과: 사용자 입력 수만큼 (기본 10명)
- 관찰 결과: 사용자 입력 수만큼 (기본 10회)
- 체험 결과: 사용자 입력 수만큼 (기본 10회)

### 시뮬레이션 품질 기준
- **현실성**: 실제 고객이 겪을 법한 구체적인 상황과 감정 기술
- **다양성**: 응답자 간 의견/경험의 편차 존재 (만장일치 금지)
- **일관성**: 동일 인물의 응답 내 모순 없음
- **구체성**: 추상적 표현 대신 구체적 행동/시간/장소/도구 기술
- **부정 응답 포함**: 전체의 20-40%는 부정적/비판적 관점 포함

## 주의사항

- 각 조사 결과는 구체적이고 현실적이어야 함
- 실제 고객의 언어와 표현 사용
- 정성적 데이터와 정량적 데이터 균형
- Journey Map 작성에 바로 활용 가능하도록 상세하게 작성
- 고객경험 단계별로 데이터 구조화
- Pain Point와 니즈를 명확히 구분
- 정량적 지표 (만족도, 빈도) 포함
