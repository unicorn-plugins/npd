# 고객경험 단계 정의 가이드

## 목적

MVP 주제와 고객유형을 기반으로 현재 고객이 겪는 경험 단계를 정의하여,
이후 고객경험 조사와 여정 맵 작성의 기준을 제공함.

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| MVP 정의 | `docs/plan/define/MVP정의.md` | 서비스 유형에 맞는 경험 단계 패턴 선택 |
| 고객 분석 | `docs/plan/define/고객분석.md` | 타겟 고객의 행동 패턴을 기반으로 단계 도출 |
| 시장 조사 | `docs/plan/define/시장조사.md` | 시장 트렌드와 고객 행동 변화를 단계에 반영 |

## 방법론

### 1. 고객 행동 흐름 분석

고객이 문제를 인식하고 해결하기까지의 전체 흐름을 파악함:
- 문제 인식 시점
- 해결 방법 탐색
- 정보 수집
- 의사 결정
- 실행/사용
- 평가/관리

### 2. 경험 단계 도출 패턴

**정보/지식 관리 서비스**
```
니즈 인식 -> 방법 탐색 -> 수집 -> 분류/정리 -> 활용 -> 최적화/관리
```

**제품/서비스 구매**
```
니즈 발생 -> 정보 탐색 -> 비교/평가 -> 구매 결정 -> 사용 -> 재구매/추천
```

**일상 생활 서비스**
```
필요 인식 -> 옵션 탐색 -> 시도/체험 -> 습관화 -> 최적화 -> 유지/관리
```

**여행/경험 서비스**
```
영감/동기 -> 계획/준비 -> 예약 -> 경험 -> 기록/공유 -> 회상
```

### 3. 단계별 특징 요건

각 단계는 다음을 충족해야 함:
- **명확성**: 단계명이 고객 행동을 명확히 설명
- **순차성**: 논리적인 흐름으로 연결
- **완결성**: 문제 인식부터 해결/관리까지 전체 커버
- **적절성**: 5-7개 단계로 구성 (너무 세분화하거나 추상적이지 않게)

## 출력 형식

```markdown
# 고객경험 단계 정의

## MVP 주제
{MVP 주제}

## 고객유형
{고객유형}

## 고객경험 단계
\```
{단계1} -> {단계2} -> {단계3} -> {단계4} -> {단계5} -> {단계6}
\```

## 단계별 설명

### {단계1}
- 주요 행동: {설명}
- 좋은 생각/느낌: {설명}
- 주요 Pain Points: {설명}

(각 단계별 반복)

## 단계 도출 근거
- {근거1}
- {근거2}
```

## 품질 기준

### 완료 체크리스트
- [ ] 고객경험 단계 5-7개 도출
- [ ] 각 단계별 주요 행동/생각/Pain Points 작성
- [ ] 단계 간 순차성(논리적 흐름) 확인
- [ ] 문제 인식부터 해결/관리까지 전체 커버
- [ ] 고객 관점의 행동 중심으로 작성됨 (기업 프로세스 아님)
- [ ] 단계 도출 근거 제시
- [ ] MVP정의.md, 고객분석.md와 일관성 확인

### 정량 기준
- 고객경험 단계: 5-7개
- 각 단계: 행동 + 생각/느낌 + Pain Points 3가지 필수

## 주의사항

- 너무 세분화하지 말고 5-7개 단계로 간결하게
- 고객 관점의 행동 중심으로 작성 (기업 프로세스가 아님)
- 단계는 화살표(->)로 연결
- 이후 모든 고객경험 조사는 이 단계를 기준으로 수행
- 고객 여정 맵 작성 시 X축으로 사용
