# 문제 가설 정의 가이드

## 목적

고객 여정 분석과 고객 경험 조사 데이터를 기반으로 핵심 문제 가설과 근본원인을 도출하고,
문제검증 인터뷰를 통해 문제가설을 검증함. 그리고 비즈니스 가치를 도출함.

## 입력 (이전 단계 산출물)

| 산출물 | 파일 경로 | 활용 방법 |
|--------|----------|----------|
| 고객 분석 | `docs/plan/define/고객분석.md` | 고객 세그먼트별 문제 인식 차이 분석에 활용 |
| 유저 저니 맵 | `docs/plan/define/유저저니맵.md` | Pain Points에서 현상문제 도출, 감정 저점 파악 |
| 관찰 결과 | `docs/plan/define/관찰결과.md` | 현상문제 도출의 근거 데이터로 활용 |
| 체험 결과 | `docs/plan/define/체험결과.md` | 현상문제 도출의 근거 데이터로 활용 |
| 인터뷰 결과 취합 | `docs/plan/define/고객경험인터뷰결과취합.md` | 핵심 인사이트에서 문제 패턴 도출 |

## 방법론

### 1. 문제 가설과 근본원인 도출

가장 큰 현상문제 3개를 도출하고 5WHY로 근본원인 도출함.

#### 5WHY 작성 형식

| 단계 | 질문 및 응답 |
|:----:|-------------|
| **WHY1** | 왜 {현상}인가? → {응답} |
| **WHY2** | 왜 {WHY1 응답}인가? → {응답} |
| **WHY3** | 왜 {WHY2 응답}인가? → {응답} |
| **WHY4** | 왜 {WHY3 응답}인가? → {응답} |
| **WHY5** | 왜 {WHY4 응답}인가? → {응답} |

- WHY1은 현상문제에 대한 질문 + 질문에 대한 응답으로 작성
- WHY2~WHY5는 이전 WHY의 응답에 대한 질문 + 질문에 대한 응답으로 작성

#### 다층적 근본원인 검토

근본원인은 아래 4가지 측면으로 검토하여 가장 중요한 측면 한개로 근본원인을 식별함:

| 측면 | 검토 항목 |
|------|----------|
| **사용자 심리적** | 인지 부하, 감정, 동기, 습관, 저항감 |
| **제도/시스템적** | 정책/규정, 프로세스, 권한 구조, 인센티브, 예산/자원, 기존 시스템 제약 |
| **맥락적** | 조직 문화, 업계 관행, 시장 상황, 타이밍 |
| **커뮤니케이션** | 정보 전달, 교육, 피드백 체계 |

#### 핵심 근본원인 식별 형식

| 측면 | 근본 원인 | 중요도 |
|------|----------|:------:|
| 사용자 심리적 | {근본원인} | ★★★★☆ |
| 제도/시스템적 | {근본원인} | ★★★☆☆ |
| 맥락적 | {근본원인} | ★★★★★ |
| 커뮤니케이션 | {근본원인} | ★★★☆☆ |

### 2. 문제가설 검증

#### 문제검증 인터뷰
- 대상자별 문제검증인터뷰 결과를 개별적으로 생략하지 않고 모두 제공
- 각 개별 인터뷰 결과에 문제 가설 3개의 중요도와 불편도를 5점 만점으로 평가한 결과 포함
- 각 문제의 근본원인에 대한 동의 여부도 조사

**인터뷰 결과 형식:**
```markdown
## 인터뷰 결과지 N
**인터뷰 일시:** {날짜}
**장소:** {장소}

### 인터뷰 대상
- **이름:** {이름}
- **성별:** {성별}
- **연령대:** {연령대}
- **직업:** {직업}

### 문제 검증
**문제 1: {문제}**
- 중요도: {점수}/5
- 불편함 정도: {점수}/5
- 근본원인에 대한 동의여부: {의견}

### 고객 관점 문제
{고객이 제시한 더 중요한 문제}

### 개선 제안
{고객의 개선 아이디어}
```

#### 문제검증 인터뷰 결과 취합 형식
```markdown
## 문제별 중요도/불편도 평가
| 항목 | 문제1 중요도 | 문제1 불편도 | 문제2 중요도 | 문제2 불편도 | 문제3 중요도 | 문제3 불편도 |
|------|------------|------------|------------|------------|------------|------------|
| 평균 | {점수} | {점수} | {점수} | {점수} | {점수} | {점수} |
```

### 3. 비즈니스 가치 도출

근본원인을 제거하여 문제를 해결했을 때 고객과 회사가 얻게 되는 비즈니스 가치를 각각 3개 이하로 도출함.

```markdown
## 고객측면의 비즈니스 가치
- {비즈니스 가치 1}
- {비즈니스 가치 2}
- {비즈니스 가치 3}

## 회사측면의 비즈니스 가치
- {비즈니스 가치 1}
- {비즈니스 가치 2}
- {비즈니스 가치 3}
```

## 품질 기준

### 완료 체크리스트
- [ ] 핵심 문제 3개 도출
- [ ] 각 문제별 5WHY 완성 (최소 5단계)
- [ ] 다층적 근본원인 4관점(심리적/제도/맥락적/커뮤니케이션) 검토
- [ ] 문제검증 인터뷰 결과 개별 제공 (종합 요약만 하지 않음)
- [ ] 비즈니스 가치 도출 (고객/회사 각 3개 이하)
- [ ] 유저저니맵의 Pain Points와 연결 확인
- [ ] 문제는 고객 관점에서 작성, 솔루션 미포함

### 정량 기준
- 핵심 문제: 3개
- 5WHY: 문제당 5단계
- 근본원인 검토: 4가지 측면
- 문제검증 인터뷰: 사용자 입력 수만큼 (기본 10명)
- 비즈니스 가치: 고객/회사 각 3개 이하

## 주의사항

- 문제는 고객 관점에서 작성
- 솔루션은 포함하지 않음
- 5WHY는 최소 3단계 이상
- 인터뷰 결과를 종합 요약만 하지 않을 것
- 모든 대상자의 인터뷰 결과 개별 제공
